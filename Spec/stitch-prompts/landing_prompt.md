# Landing Prompt

## Base Prompt

Create a responsive web landing page for a product called Chess Trainer.

This is a signed-out public homepage.

The product imports chess games, analyzes mistakes, and turns them into training puzzles.

The design direction is a workbench showcase, not generic SaaS marketing.

The first screen must have:

- one large chess board or analysis-workbench visual
- one short headline
- one primary Google sign-in CTA
- one secondary workflow CTA

Include sections for:

- import
- analysis
- puzzle generation
- personal training value

Use product cues like eval graphs, move lists, and puzzle cards as polished previews, not as a live dashboard.

The page should feel:

- premium
- restrained
- chess-specific
- readable on desktop and mobile

Avoid:

- cards everywhere
- logo clouds
- generic startup illustrations
- generic purple SaaS styling
- cluttered hero sections

The board and workbench language should be the main visual story.

## Refinement Prompts

### Refinement 1

Make the board and eval graph larger and reduce generic marketing chrome.

### Refinement 2

Reduce the copy by 30 percent and make the primary CTA more obvious.

### Refinement 3

Make the first screen feel more like a chess study poster and less like a SaaS homepage.

### Refinement 4

Keep the page brighter and cleaner, with one warm accent color only.

## Success Check

The result is good if:

- the product is obviously about chess in the first screen
- the workbench is the hero, not decoration
- the CTA is immediately clear
- the page feels like one strong composition, not a grid of unrelated sections
